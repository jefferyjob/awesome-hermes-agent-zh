# 参与贡献 Awesome Hermes Agent

感谢你愿意参与贡献！这份列表旨在成为一个高质量、经过精心筛选的 Hermes Agent 生态资源合集。

## 如何推荐资源

请**[创建一个 issue](https://github.com/jefferyjob/awesome-hermes-agent-zh/issues/new)**，并附上以下信息：

- **资源名称**及 URL
- **作者**（GitHub 用户名或姓名）
- **简要描述**其用途
- **所属分类**（分类见 README）
- **为什么它值得收录** —— 它的亮点是什么？

请不要直接通过 PR 添加资源。所有新增内容都会由维护者审核并添加，以确保整体质量和编辑风格一致。

## 质量标准

资源应满足以下标准：

1. **相关性** —— 与 Hermes Agent 或 agentskills.io 生态直接相关
2. **文档完善** —— 有清晰的 README，说明其功能和使用方式
3. **可用** —— 确实可以运行（或已明确标注为实验性 / 开发中）
4. **非重复** —— 不与列表中已有资源重复
5. **开源** —— 源代码公开可用
6. **有维护** —— 优先收录近期仍有活动的项目（或虽较早但仍具基础价值的项目）
7. **为什么是现在** —— 请用一句话说明该资源为什么对当前 Hermes 用户有意义，而不只是泛泛而谈

## 成熟度标签

列表中的每个资源都会标注一个成熟度等级：**production**、**beta** 或 **experimental**。这些标签由维护者根据文档质量、维护活跃度和社区采用情况进行判定。

如果你认为某个标签不准确 —— 例如某个项目自收录以来已经更成熟，或者某个标为 “production” 的项目已经停滞 —— 请[创建一个 issue](https://github.com/jefferyjob/awesome-hermes-agent-zh/issues/new)告诉我们。相比让人把一个周末浪费在尚未准备好的项目上，我们更愿意及时更新标签。

## 关于仓库本身的建议

如果你想对这个 awesome 列表本身提出建议（例如分类、格式、结构），请[创建一个 issue](https://github.com/jefferyjob/awesome-hermes-agent-zh/issues/new)描述你的修改建议。

## 行为准则

本项目遵循 [Contributor Covenant 行为准则](CODE_OF_CONDUCT.md)。参与本项目即表示你同意遵守其中条款。
